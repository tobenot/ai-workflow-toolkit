# notify-done 新系统故障说明与修复方案（可直接发送）

各位同事好，

近期我们在迁移到新系统后发现 `notify-done` 提示音脚本出现异常，表现为脚本启动后在配置加载阶段报语法错误，导致提醒链路中断。经排查，问题并非业务逻辑变更，而是**运行环境差异触发了脚本兼容性缺陷**。

本次问题的根因主要有两点：

1. `notify-done.config.ps1` 中包含中文字符串字面量，在新系统的 PowerShell 读取路径下发生编码解析偏差，触发 parser error；
2. 子进程启动逻辑硬编码为 `powershell.exe`，在新环境（宿主/路径策略变化）下存在兼容风险。

修复方案已落地：

- 将配置中的中文默认文案改为 **Base64 + 运行时 UTF-8 解码**，规避源码编码差异；
- 将子进程启动改为**动态解析当前 PowerShell 可执行路径**，取消宿主硬编码。

验证结果：已在新系统完成入口模式与 worker 直连模式（`-Worker -NoPopup -NoBeep`）实测，脚本均正常执行并成功收尾。后续建议将“跨系统脚本兼容回归”加入变更检查项，避免同类问题再次出现。
